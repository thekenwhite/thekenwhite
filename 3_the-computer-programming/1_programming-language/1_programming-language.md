
# Programming Language

컴퓨터가 동작하게 되는 원리는, 간단하게 말하면 명령어와 데이터다.

예를 들어 다음과 같은 조합을 CPU에 넘겼다고 가정한다.

- 0101 0001 0001 

이때 CPU는 처음에 있는 0101은 더하라는 명령어로 해석한다.

그리고 0001과 0001을 더한다.

더하면 답은 0010이 된다.

이때 더하라는 명령어가 0101인지 1010인지는 하드웨어 제조사에 따라 다를 수 있다.

그리고 그걸 ADD 같은 최소한의 추상성으로 묶는 것이 바로 어셈블리어다.

그리고 사실 저런 식으로 직접적으로 값을 넘기지도 않는다.

어디 저장할 데가 있어야 할 것 아닌가.

실제로는 이런 느낌으로 작동한다.

- 0x0101이라는 주소에 1 저장
- 0x0110이라는 주소에 1 저장
- 0101 0x0101 0x0110
	- 0x0101이라는 주소에 있는 값에 0x0110의 주소에 있는 값을 더하고 그대로 저장해라
	- (또는 그 반대)
- 연산 이후 0x0101은 2가 되고, 0x0110은 1로 남는다.

그런데 이런 어셈블리어를 사람이 일일이 치는 건 상당히 머리가 좀 아플 수 있다.

그래서 사람이 좀 쓸 만하게 포장해 놓은 것을 고급 프로그래밍 언어, 또는 그냥 일반적으로 프로그래밍 언어라고 한다.

다만 그래서 '어떻게' 포장을 할 건지에 대한 철학이 프로그래밍 언어마다 좀 다르다.

## The C Programming Language

C는 프로그래밍 언어 중에서 상당히 원시적인 프로그래밍 언어에 속한다.

여러 프로그래밍 언어 중에선 꽤나 초기에 속하는 언어인 데도 불구하고, 아직까지 쓰이고 있고, 또 다른 프로그래밍 언어에게도 프로그래밍 언어가 나아가야 할 길을 제시해줬다는 평가를 받는다.

C 프로그래밍 언어는 Unix 운영체제의 등장과 관련이 깊다.

C는 Unix라는 운영체제 자체를 만드는 데에도 사용되었지만, Unix 위에서 돌아가는 애플리케이션을 만드는 데에도 사용되었다.

Unix-like 운영체제는 C에 필요한 헤더파일, 라이브러리 파일 등을 시스템에 저장해 놓는다.

그러니까 그게 좀 더 네이티브한 C의 사용법이다.

헤더파일이나 라이브러리의 경로를 일일이 지정하게 하는 Windows가 특이한 것이다.

## The C++ Programming Language

C++는 C를 기반으로 하고 있는 언어다.

C가 좀 더 시스템 프로그래밍 언어의 느낌이 강하다면, C++는 그 C를 사용하여 응용 프로그램을 만드는 데에 좀 더 특화되었다는 느낌이 있다.

C 문법을 기반으로 한다는 점에 C++가 단순히 더 기능이 많은 언어라고 생각하기 쉽지만, 사실 C와 C++가 가는 길은 상당히 다르다.

C는 기본적으로 정적이고 보수적인 느낌이다. 아무리 시간이 지나도 변화가 잘 일어나지 않는다.

반면 C++는 3년마다 표준이 개정되고, 표준이 개정될 때마다 지향하는 방향도 상당히 달라진다.

그래서 같은 프로그래밍 언어를 쓰는 데도 쓰는 사람의 습관, 스타일, 철학에 따라 코드 느낌이 상당히 많이 달라질 수 있다.

이 점을 좋아하는 C++ 프로그래머도 있지만, 이 점을 싫어하는 C 프로그래머도 있다.

C++의 사용은 특히 Windows에서 유명하다.

Unix 스타일은 C++ 확장자를 `.cc`로 사용하는데, MS 스타일은 `.cpp`로 사용한다. 그런데 사람들의 파일 확장자를 보면 cpp의 사용이 더 많다는 것을 알 수 있다.

이는 C++ 생태계에 MS가 끼친 영향이 지대했음을 의미한다.

따라서 이렇게 생각하면 된다.

- Unix 쪽은 C를 좀 더 선호한다.
- MS 쪽은 C++를 좀 더 선호한다.

MS의 C++은 그냥 C++가 아니다. Visual C++라고 해서, 좀 더 개조가 진행되어 있다. Microsoft Visual C++라고 해서, 줄여서 MSVC라고도 한다.

VC++만 쓰고 있으면 그것이 당연히 C++에 있는 기능이라고 생각하기 쉽다. 그러나 사실은 VC++에만 있는 기능이었던 경우도 많다.

## Other Programming Languages

C, C++ 외에도 프로그래밍 언어는 잔뜩 있다. JS는 원래 웹 프론트엔드 언어였다가, 노드 같은 것들 때문에 백엔드에서도 쓸 수 있게 된 언어다. JS는 배우는 양을 줄이고 싶어하는 웹 입문자들이 많이 사용하는 언어다.

또, 프로그래밍 입문을 할 때 Python이라는 언어도 자주 사용된다. Python은 일반적으로 사람들이 가장 쓰기 편하다고 느끼는 언어로 알려져 있다. 성능 효율성이 그렇게 좋은 것은 아니나, 일단 코딩이라는 것을 체험해 보게 할 때 자주 쓰는 언어다.

그렇다고 해서 마냥 입문자용 언어인 것은 아니다. 코드 양을 비교적 적게 쳐도 된다는 점 때문에, 일단 빨리 빨리 개발해서 어떤 결과를 보고 싶어하는 경우 Python을 자주 쓰기도 한다. 그래서 Python은 뭔가를 할 수 있는 라이브러리도 많다. 단순히 기능적인 측면에서 무언가를 할 수 있게 해주는 라이브러리가 아니라, 당장 어떤 결과물을 보는 데 사용할 수 있는 라이브러리다. 그래서 다른 프로그래밍 언어와는 상당히 느낌이 다르다.

Java는 플랫폼 간의 차이를 최대한 줄이기 위해 나온 언어다. 특히 기업이 선호하는 경향이 있다. 한국의 경우 유독 Java 사용이 심한 경향이 있다. 특히 또 Spring이라는 프레임워크를 많이 사용한다. JS가 요즘 입문하려는 사람들이 백엔드로 자주 쓰려는 언어라면, Java는 예전부터 웹 업계에서 일해오던 사람들이 자주 쓰는 언어다. 또 JSP라고 해서, JS처럼 웹 페이지 형식에다가 Java 코드를 사용하는 것도 가능하다.

웹에서 쓰는 것은 PHP, ASP 등도 존재한다. 예전부터 있어왔던 웹 페이지가 PHP를 쓰는 경우가 많다. 예를 들어 디시인사이드도 PHP를 쓰고, 페이스북도 한때 PHP를 썼었다. 워드프레스라는 CMS도 PHP를 쓰는 것으로 유명하다. 하지만 PHP는 버그가 유독 자주 발생하는 언어라서, 사실 PHP를 싫어하는 개발자들도 많다. PHP에 관한 유명한 농담이 있다. 폰허브라는 야동 사이트에서 PHP 구인을 한 적이 있는데, 거기서 댓글로 이런 것들이 달린 적이 있다.

- 흥미롭긴 한데 주변 사람들한테 내가 뭘 하는지 쉽게 말할 수 있을 것 같진 않다.
- 이해한다. 나 같아도 다른 사람들한테 PHP를 쓴다고 차마 말할 수 없을 것 같다.

PHP는 대략 그런 취급을 받고 있는 언어라고 생각하면 된다. 페이스북 같은 경우 PHP를 버리고 React라는 프레임워크를 만들었고, 또 Hack이라는 언어도 만들었다.

React, Vue, Angular, 이런 것들은 전부 JS를 쓰는 것들이다. 또 백엔드 JS 같은 경우도 노드 외에도 Deno, Bun 같은 것들도 언급되고 있다. JS도 버그가 많다는 이미지가 있는데, JS는 웹 생태계에 너무 뿌리 깊게 박혀서 이젠 빼내기가 좀 상당히 힘든 언어다. 그래서 최소한 TS, TypeScript라는 걸로 포장하려는 시도도 좀 있고, 최근 같은 경우에는 WebAssembly 같은 것들이 JS의 대안으로 제시되고 있기도 하다.

프로그래밍 언어는 그밖에도 많다. C# 같은 경우 마이크로소프트의 Java라는 소리를 듣고 있는 언어다. Windows 애플리케이션을 C++ 말고도 C#으로 개발하는 것도 가능하다. 말이 C 들어간 C#이지, Unix와 C#은 별로 친하지 않다. C#의 우선 지원 대상은 당연히 MS Windows다. 그밖에도, Ruby, R, 뭐 별 게 다 있다. 예전부터 내려오던 언어로는 포트란, 코볼 같은 것도 존재한다.

요즘 프로그래밍 메타는 함수형이다. 왜냐하면 함수형이 컴퓨터라는 것의 본질에 가장 잘 맞는다는 평가를 받고 있기 때문이다. 특히 비판받는 것이 바로 OOP(Object-Oriented Programming), 객체 지향 프로그래밍이다. C++, Java, C# 등이 객체 지향 프로그래밍 언어다.

하지만 무슨 프로그래밍 언어든 간에 다 애플리케이션 레벨 정도에서만 노는 느낌이 있었다. 시스템 프로그래밍, 로우레벨 프로그래밍 영역은 결국 계속해서 C, C++가 장악하고 있는 건가 하는 인식이 팽배해 있었다. 그러다가 그 틀을 깨게 된 것이 바로 러스트다.

## The Rust Programming Language

러스트는 비교적 신생 언어다. 최초 출시가 2015년이다. 이후 3년마다 개정을 발표하고 있다.

러스트가 특이한 이유는, 프로그래밍이란 것에 대해 완전히 색다른 접근법을 가지고 있기 때문이다.

러스트의 기본 정신은 일단 복사를 함부로 하지 않는 것이다.

함수에 무언가를 넘기는 행위는 전부 기본적으로 이동이다. 기본 행동이 복사인 다른 프로그래밍 언어와는 사뭇 다른 모습이다.

이 때문에 러스트는 차라리 처음 배운다면 더 잘 배울 수도 있는 언어다. 다른 프로그래밍 언어가 습관이 된 사람들은 러스트의 방식이 다소 익숙치 않을 수도 있다.

그리고 다른 프로그래밍 언어 같은 경우, 특히 C나 C++에서 자꾸 복사를 방지하기 위해서 이래라 저래라 권장하는 게 많은데, Rust는 그걸 기본적으로 수행하고 있기 때문에 훨씬 생각하기가 편하다.

그래서 차라리 Rust에서 프로그래밍 습관을 잘 잡고, 필요하면 그 습관을 가지고 가서 다른 프로그래밍 언어를 하는 게 더 낫다고 본다. 일단 막 짜려고 하게 되는 초기 프로그래머 특성상, 다른 언어를 쓰려고 했다간 습관 잘못 잡히기 딱 좋다. 러스트는 상당히 빡빡한 룰을 통해서 프로그래머가 엇나가지 않게 해준다.

내가 주로 주목하고 싶은 프로그래밍 언어도 러스트다. C가 초기에 프로그래밍의 방향을 제시해준 것처럼, 러스트가 앞으로의 프로그래밍의 방향을 제시하고 있다고 생각하기 때문이다.

그러나 일단 순서는 순서이기 때문에, C랑 C++는 한번 짚고 넘어가도록 하겠다.
